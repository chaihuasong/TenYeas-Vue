## 三、关键优化点

### 1. Loading 状态

**优化前：** 数据加载时页面无反馈，用户不知道是否在请求中

**优化后：**
```javascript
// data 中添加 loading 状态
data() {
  return {
    loading: false,
    // ...
  }
}

// 请求时设置 loading
getData() {
  this.loading = true
  axios({...}).then((res) => {
    this.tableData = res.data
  }).finally(() => {
    this.loading = false
  })
}
```

```html
<!-- 表格添加 v-loading 指令 -->
<el-table :data="tableData" v-loading="loading">

<!-- 分页器加载时禁用 -->
<el-pagination :disabled="loading">
```

### 2. 错误处理

**优化前：** API 请求失败时静默失败，用户无感知

**优化后：**
```javascript
axios({...})
  .then((res) => {
    // 成功处理
  })
  .catch((err) => {
    console.error('获取数据失败:', err)
    this.$message.error('获取数据失败，请重试')
  })
  .finally(() => {
    this.loading = false
  })
```

### 3. 分页修复 (DailyReportList.vue)

**优化前：** 切换分页时不重新请求数据
```javascript
handleSizeChange(val) {
  this.pageSize = val
  // this.getData()  // 被注释掉了
}
```

**优化后：**
```javascript
handleSizeChange(val) {
  this.pageSize = val
  this.getData()  // 切换每页条数时重新请求
}

handleCurrentChange(val) {
  this.currentPage = val
  this.getData()  // 切换页码时重新请求
}
```

### 4. 删除索引修复

**优化前：** 删除时使用错误的索引
```javascript
delItem(index) {
  // ...
  this.tableData.splice(index, 1)  // 错误：没考虑分页偏移
}
```

**优化后：**
```javascript
delItem(index) {
  let id = (this.currentPage - 1) * this.pageSize + index  // 计算实际索引
  // ...
  this.tableData.splice(id, 1)  // 正确：使用计算后的索引
}
```

### 5. 删除逻辑简化

**优化前：** 删除后再查询一次验证是否删除成功
```javascript
axios.post('delete', data).then((res) => {
  axios.get('getById?id=' + id).then((res) => {  // 不必要的二次查询
    if (res.data != null) {
      this.$message.warning('删除失败！')
    } else {
      this.$message.success('已删除！')
    }
  })
})
```

**优化后：** 直接根据删除请求结果处理
```javascript
axios.post('delete', data)
  .then(() => {
    this.$message.success('已删除！')
    this.tableData.splice(id, 1)
  })
  .catch((err) => {
    console.error('删除失败:', err)
    this.$message.error('删除失败，请重试')
  })
```

## 七、后续优化建议

1. **添加请求取消** - 快速切换页面时取消未完成的请求
2. **添加数据缓存** - 对不常变化的数据（如模板列表）进行缓存
3. **搜索防抖** - 输入搜索关键词时添加防抖，减少请求次数
4. **统一 API 层** - 将分散的 axios 调用封装到统一的 API 服务层
